# Pinterest Command Debug & Fix Skill

## Purpose
This skill provides a structured approach for diagnosing and fixing the `-pint` command when it fails to return Pinterest images.

## Problem Statement
The `-pint` command sometimes returns "no results" despite valid search queries. Common causes:
1. Pinterest HTML structure changed (regex patterns don't match)
2. Network/SSL errors (axios request fails)
3. Rate-limiting by Pinterest
4. Bing Images fallback not triggered or also fails
5. URL extraction logic misses valid pinimg URLs

## Prerequisites
- Project setup: `npm install` completed
- `.env` configured with valid LLM key
- Baileys authentication: `baileys_auth/` folder present
- Network access: Can reach pinterest.com and bing.com

## Common Fixes

### Fix 1: Update Regex Patterns
If Pinterest HTML structure changed, regex patterns may not match current structure.

**Current patterns (line ~2765-2770):**
- `directRegex`: Matches `https://i.pinimg.com/...` directly in HTML
- `escapedRegex`: Matches `https:\/\/i\.pinimg\.com\/...` (backslash-escaped)
- `jsonUrlRegex`: Matches `"url":"https:\/\/i\.pinimg\.com\/..."` in JSON

**To update:**
1. Open Pinterest in browser: https://www.pinterest.com/search/pins/?q=Manali
2. Right-click → Inspect → Network tab → Reload
3. Find response with large JSON payload (contains image URLs)
4. Search for `pinimg` URLs in response
5. Update regex to match new format
6. Test: Run `-pint Manali` again

### Fix 3: Handle Network Errors
Wrap axios call in better error handling:

```javascript
try {
  const response = await axios.get(searchUrl, { ... });
  // ... rest of logic
} catch (error) {
  console.error("DEBUG: Axios error:", error.code, error.message);
  if (error.code === 'ECONNREFUSED') {
    return "❌ Network error - can't reach Pinterest. Try again later.";
  }
  if (error.response?.status === 429) {
    return "❌ Pinterest rate-limited. Wait a few minutes.";
  }
  // ... other error types
}
```

## Performance Considerations

- **Timeout:** axios set to 20s; Pinterest may take 15-19s to respond
- **Image Processing:** Sharp resizes images before sending (slow on Raspberry Pi)
- **Concurrent Requests:** Each -pint call is sequential; multiple calls wait for previous
- **URL Normalization:** Regex + string replacements are CPU-bound; ~50ms per 100 URLs

## Prevention & Best Practices

1. **Monitor Pinterest Changes:** If scraping stops working, Pinterest likely changed HTML/API
   - Set calendar reminder: Check -pint quarterly
   - Keep backup of regex patterns in comments

2. **Use Async Fallbacks:** Always have Bing or alternative image source as backup
   - Current: Pinterest primary → Bing secondary
   - Future: Consider Google Images or Pinterest API (if available)

3. **Error Logging:** Always log specific error stages for faster debugging
   - Network error vs parsing error vs validation error

4. **Rate Limit Handling:** Add exponential backoff for 429 responses
   - Respect Pinterest's rate-limiting

5. **URL Caching:** Cache successfully extracted URLs to reduce repeat requests
   - Within same session (5-10 min TTL)
